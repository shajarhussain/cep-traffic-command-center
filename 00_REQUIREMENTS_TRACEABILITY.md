## Minimum demo flow for evaluation
1. Start backend and frontend (`npm run dev`).
2. Open Command Center (default route). The KPI strip, hero header, and Weather Context card all populate from live data.
3. **Subscribers Monitor** (System → Subscribers): show all five subscribers (Alert, Logging, Dashboard, Reporting, Incident) and their currently subscribed event types. Quote the UI's footer card explaining the Observer Pattern.
4. **Simulator** (Cameras → Event Simulator): publish a `VehicleDetectedEvent`; switch back to Command Center to see the KPIs update; switch to Intersection Intelligence to see the dashboard snapshot.
5. **Speed violation + idempotency** (Cameras → Event Simulator → Duplicate Alert Safety): use the "Run Duplicate Test" button, then go to Enforcement → exactly one penalty. The Fine Policies tab in Config Center shows the user-driven fine schedule that produced the amount.
6. **Subscribe / unsubscribe demo** (System → Subscribers): click × on the `SpeedViolationEvent` chip under AlertService. Publish a SpeedViolation from the simulator → no penalty. Click **Restore Defaults** and re-publish → penalty appears.
7. **Bounded queue + flood test** (System → Capacity → Stress Test the Bounded Queue): preset 500 → Run. See accepted/evicted breakdown by event type. Compare with the 23.81s calculation above the gauge.
8. **Outbox Pattern** (System → Outbox): publish a few events from the simulator; rows appear with status PUBLISHED. Kill the API mid-publish; the next published row stays PENDING. Restart and rows auto-publish from the background relay.
9. **External integrations** (Command Center hero): TomTom connection pill turns green once the key is configured in `.env` and the provider is enabled in Configuration Center. Weather Context populates automatically from Open-Meteo (no key needed).
10. Walk through UML diagram and ADRs in `05_DESIGN_PATTERNS.md` and `07_CLO4_ANALYSIS_ADR.md`.
